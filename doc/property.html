property

handlers property

Source:
  • jquery.aria.js, line 691

Handles WAI-ARIA properties without modifying the values any more than it needs to. These methods also act as the fallback for other namespaces such as handlers.reference and handlers.state.
handlers.property.parse parses the attribute name.
handlers.property.get gets the value of the property.
handlers.property.set sets a property.
handlers.property.has checks to see if the property exists.
handlers.property.unset removes the property.

Methods

(private, static) get(element, name) → {String|undefined}

Source:
  • jquery.aria.js, line 888

Gets the value of the WAI-ARIA property from the given element and returns it unchanged. The name is normalised (see jQuery.normaliseAria). If element is not an element (see isElement) or name is not recognised (see handlers.property.has) then undefined is returned.

Example
// Markup is:
// <div id="one" aria-label="test"></div>

var element = document.getElementById("one");
handlers.property.get(element, "label"); // -> "test"
handlers.property.get(element, "busy"); // -> undefined

(private, static) has(element, name) → {Boolean}

Source:
  • jquery.aria.js, line 850

Checks to see if the given name exists on the given element. The name is always normalised (see jQuery.normaliseAria) and if element is not an element (see isElement) then false will always be returned.

Example
// Markup is:
// <div id="one" aria-label="test"></div>

var element = document.getElementById("one");
handlers.property.has(element, "label"); // -> true
handlers.property.has(element, "busy");  // -> false

(private, static) set(element, name, value, indexopt, convertopt)

Source:
  • jquery.aria.js, line 788

Sets the property of an element. The value is unchanged (other than normal string coercion) and the name is normalised into a WAI-ARIA property (see jQuery.normaliseAria).

If element is not an element (see isElement) then no action will be taken.

If value is a function, it is treated like an Attribute_callback. This is for consistency with jQuery#attr.

A convert function can also be passed. That function will convert value (if value is a function, convert will convert the result) before assigning it. If convert is ommitted or not a function then identity is used so value will not be changed.

Examples

Setting a property

// Markup is:
// <div id="one"></div>

var element = document.getElementById("one");
handlers.property.set(element, "label", "test");

// Now markup is:
// <div id="one" aria-label="test"></div>

Setting a property using a function

// Markup is:
// <div id="one" aria-label="test"></div>

var element = document.getElementById("one");
handlers.property.set(element, "label", function (i, attr) {
    return this.id + "__" + i + "__" + attr;
}, 0);

// Now markup is:
// <div id="one" aria-label="one__0__test"></div>

Converting the result

// Markup is:
// <div id="one" aria-label="test"></div>

var element = document.getElementById("one");
handlers.property.set(element, "label", function (i, attr) {
    return this.id + "__" + i + "__" + attr;
}, 0, function (value) {
    return value.toUpperCase();
});

// Now markup is:
// <div id="one" aria-label="ONE__0__TEST"></div>

(private, static) unset(element, name)

Source:
  • jquery.aria.js, line 929

Removes a WAI-ARIA attribute from the given element. The name if normalised (see jQuery.normaliseAria) and if element is not an element (see isElement) then no action is taken.

Example
// Markup is:
// <div id="one" aria-label="test"></div>

var element = document.getElementById("one");
handlers.property.unset(element, "label");

// Now markup is:
// <div id="one"></div>
